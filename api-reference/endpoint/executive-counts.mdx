---
title: 'Executive Counts'
api: 'POST /enrichment-service/executive-counts'
---

Returns per-role-group and per-status officer counts for a list of SIRENs — **without** triggering enrichment or charging contact credits. Use this to populate filter chips (with counts) on the bulk-upload UI before the user starts the enrichment job.

## Request Body

<ParamField body="data" type="object">
  <Expandable title="properties">
    <ParamField body="siren_numbers" type="string[]" required>
      List of SIREN numbers to count executives for.
    </ParamField>
  </Expandable>
</ParamField>

## Response

| Field | Type | Description |
|-------|------|-------------|
| `total` | integer | Total number of officers across all SIRENs |
| `role_groups` | object | Counts per role group: `Executives`, `Governance & Audit`, `Shareholders & Other`, `Liquidator` |
| `status` | object | Counts per status: `current`, `former` |
| `available_groups` | string[] | Ordered list of group names for rendering filter chips |

## Credit Cost

**No credits charged.** This endpoint only counts officers — it does not trigger enrichment.

<RequestExample>
```bash cURL
curl -X POST https://search.leads-siren.com/enrichment-service/executive-counts \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "siren_numbers": ["328015318", "123456789", "987654321"]
    }
  }'
```

```python Python
import requests

resp = requests.post(
    "https://search.leads-siren.com/enrichment-service/executive-counts",
    headers={"x-api-key": "YOUR_API_KEY"},
    json={
        "data": {
            "siren_numbers": ["328015318", "123456789", "987654321"]
        }
    }
)
print(resp.json())
```
</RequestExample>
